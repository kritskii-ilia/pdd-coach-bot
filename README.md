# PDD Coach Bot

Telegram-бот-тренажёр для подготовки к теоретическому экзамену на права РФ без внешних LLM и без привязки к OpenAI API. Проект делает упор на короткие уроки, тематические тренировки, повторение слабых мест и постепенный переход к полноценным билетам.

## Что умеет

- разделять обучение на короткие блоки теории по базовым темам;
- после урока сразу запускать тематическую тренировку;
- вести смешанные мини-тесты;
- хранить прогресс и ошибки в SQLite;
- показывать прогресс по теории, уникальным экзаменационным вопросам, билетам и дневной цели;
- возвращать слабые темы через простую адаптивную логику;
- присылать микро-касания в течение дня через `JobQueue`;
- поддерживать импорт полноценной базы билетов и изображений локальным скриптом.

## Стек

- Python 3.12
- `python-telegram-bot`
- SQLite
- JSON-контент для уроков, стартовых вопросов и плана обучения

## Структура

- `pdd_coach_bot/` — код приложения
- `content/topics.json` — темы первого этапа
- `content/lessons.json` — короткие теоретические блоки
- `content/questions_starter.json` — встроенные стартовые вопросы
- `content/study_plan_april_2026.json` — план до конца апреля
- `content/generated/` — сюда складывается импортированная база вопросов
- `assets/imported/` — изображения вопросов после импорта
- `assets/lessons/` — локальные учебные схемы и карточки по теории
- `scripts/import_external_tickets.py` — локальный импорт билетов из открытого репозитория
- `scripts/generate_lesson_slides.py` — пересборка визуальных карточек уроков
- `deploy/systemd/pdd-coach-bot.service` — systemd unit для фонового запуска в WSL
- `deploy/windows/start-pdd-coach-bot.cmd` — Windows helper для автоподъёма WSL-сервиса при входе
- `data/` — SQLite база и runtime-данные

## Быстрый запуск

1. Создай бота у `@BotFather` и получи токен.
2. Подготовь окружение:

```bash
cd /home/user/pdd-coach-bot
python3 -m venv .venv
source .venv/bin/activate
pip install -e .[dev]
cp .env.example .env
```

3. Заполни в `.env` значение `PDD_BOT_TOKEN`.
   Если Telegram API недоступен напрямую, заранее укажи proxy в `PDD_TELEGRAM_PROXY_URL`.
4. Запусти бота:

```bash
set -a
source .env
set +a
python -m pdd_coach_bot.main
```

Проверка подключения к Telegram API:

```bash
cd /home/user/pdd-coach-bot
source .venv/bin/activate
set -a
source .env
set +a
python scripts/check_telegram_connectivity.py
```

Если планируешь использовать SOCKS proxy, после обновления зависимостей переустанови проект:

```bash
cd /home/user/pdd-coach-bot
source .venv/bin/activate
pip install -e .[dev]
```

## Импорт полной базы билетов

В репозиторий билеты заранее не встроены намертво. Это сделано специально: внешний источник с вопросами и картинками не содержит явной лицензии на перераспространение, поэтому проект использует безопасный путь через локальный импорт.

Команда:

```bash
cd /home/user/pdd-coach-bot
source .venv/bin/activate
python scripts/import_external_tickets.py
```

Что делает импорт:

- клонирует `etspring/pdd_russia`;
- берёт билеты категории `A_B`;
- преобразует вопросы к внутреннему формату;
- копирует изображения в `assets/imported/`;
- создаёт `content/generated/imported_questions.json`.

После этого в боте заработает раздел `Билеты`, а тематические тренировки станут значительно богаче.

## Учебная логика

Бот не заставляет просто зубрить 40 билетов подряд.

Поток такой:

1. Базовая тема: короткий урок на 1-3 минуты.
2. Сразу закрепление: 5-7 вопросов по теме.
3. После нескольких тем: смешанный тест.
4. После базы: переход к билетам.
5. Ошибки и слабые темы получают более ранний повтор.

Статусы тем:

- `new` — не начато
- `learning` — изучается
- `weak` — усвоено слабо
- `solid` — усвоено нормально
- `strong` — усвоено хорошо

## Уведомления

Бот раз в 30 минут проверяет пользователей и в разумное окно по времени выбирает, отправлять ли ещё одно касание на сегодня. Типы касаний:

- вернуть слабую тему;
- предложить следующий короткий урок;
- прислать один вопрос дня.

Интенсивность:

- `low` — 3 касания в день
- `medium` — 5 касаний в день
- `high` — 7 касаний в день

Менять можно через раздел `Настройки`.

## Фоновый запуск и автозапуск

Для WSL с `systemd=true` подготовлен systemd-сервис:

```bash
cd /home/user/pdd-coach-bot
./scripts/install_systemd_service.sh
```

Проверка:

```bash
./scripts/service_status.sh
./scripts/service_logs.sh
```

Скрипт `install_systemd_service.sh` также ставит отдельный timer `pdd-coach-daily-plan.timer`.
Он каждые 15 минут пытается отправить дневной план и не зависит от того, успел ли встроенный scheduler бота сработать утром.

Если нужен автоподъём после входа в Windows, используется helper `deploy/windows/start-pdd-coach-bot.cmd`, который запускает WSL и даёт команду `systemctl start pdd-coach-bot.service`.

В текущей настройке helper также можно положить в папку Windows Startup:

`C:\Users\User\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\pdd-coach-bot-start.cmd`

## Источники

- Теория в уроках написана заново человеческим языком по структуре ПДД РФ.
- В качестве нормативной опоры использовались действующие Правила дорожного движения РФ, постановление Правительства РФ № 1090, в редакции, которую КонсультантПлюс показывает как действующую на 16 июля 2025 года с изменениями, вступившими в силу с 1 сентября 2025 года.
- Для локального импорта билетов предусмотрен открытый репозиторий `etspring/pdd_russia` с вопросами, пояснениями и изображениями. Импорт встроен как отдельный шаг, а не как зашитые данные.

## Ограничения текущей версии

- Встроенная стартовая база вопросов небольшая; для полноценного режима билетов нужен импорт.
- Часовой пояс пользователя пока общий по конфигу, а не настраивается через отдельный экран.
- План апреля пока задан как простой JSON-план, без перерасчёта от пропусков.
